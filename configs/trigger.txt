# ==========================================================
# Zabbix Trigger Expressions (Lab-Specific Data)
# ==========================================================

# [1] Potential Brute Force Attack Detection
# Description: Detects failed login attempts (Event ID 4625) within 1 minute.
# Logic: If any Failure Audit is recorded in the Security Log, an alert is triggered.
# Severity: High
# Target: Windows Server 2022
count(/WIN SERVER-2022/eventlog[Security,,Failure Audit,,4625,,],1m)>0

# [2] Zabbix Agent Availability
# Description: Triggers if the remote agent is unreachable for 5 minutes.
# Severity: Average
nodata(/WIN SERVER-2022/agent.ping,5m)=1

# [3] HIGH CPU UTILIZATION (Linux Server)
# Description: Cảnh báo khi CPU quá tải liên tục trong 5 phút.
# Logic: Sử dụng hàm min() để tránh báo động giả khi CPU chỉ tăng vọt tức thời.
# Dependency: Trigger này sẽ tạm ẩn nếu "Load average" đã báo động trước đó.
min(/Zabbix server/system.cpu.util,5m)>{$CPU.UTIL.CRIT}

# [4] Windows CPU Queue Length
# Description: Alert when too many processes are waiting for CPU time (Bottleneck detection).
# Logic: (System Queue Length) - (Number of Logical Processors * 2) > Threshold.
min(/WIN10/perf_counter_en["\System\Processor Queue Length"],5m) - last(/WIN10/wmi.get[root/cimv2,"Select NumberOfLogicalProcessors from Win32_ComputerSystem"]) * 2 > {$CPU.QUEUE.CRIT.MAX}

# [5] Disk Space Usage (Critical)
# Description: Alert when Drive C: is almost full (Typically > 90%).
# Logic: Monitors the percentage of used space (pused) continuously for 5 minutes.
min(/WIN10/vfs.fs.dependent.size[C:,pused],5m)>{$VFS.FS.PUSED.MAX.CRIT:"(C:)"}
